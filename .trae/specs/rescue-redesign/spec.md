# 疯友急救页面重构 Spec

## Why
当前急救页面是"发疯文案生成器"，与"死了么"的逆向逻辑不契合。需要重构为"打卡=发疯"的核心理念：来这里打卡代表你今天状态不好，需要急救/倾诉/解压。

## What Changes
- 彻底重构为"今日发疯打卡"为核心的急救页面
- 打卡代表"我今天不太好"，而非文案生成
- 视觉风格改为更像急救/医疗风格，强调"打卡求急救"的概念
- 整合紧急联系人、解压工具、专业咨询等急救功能

## Concept Design
**死了么** = "没打卡 = 出事" → 死亡打卡
**疯了么** = "打卡 = 发疯" → 急救打卡

用户来这里的心理状态：
- 今天状态不好，需要被看见
- 需要急救工具（打电话/小游戏/咨询）
- 发泄情绪的安全出口

## Impact
- Affected specs: 疯友急救中心
- Affected code: pages/rescue/index.vue, api/rescue.ts

## Design System

### 视觉风格
- 风格：急救医疗 + 赛博朋克警示
- 主色调：红/白/黑为主（急救色）
- 辅助色：警示黄、紧急红
- 图标：医疗元素（针管、药丸、心电图）+ 警告标识

### 核心交互
- 一键打卡 → 代表"我今天不太好"
- 打卡后显示急救工具箱
- 支持紧急联系人直拨

## MODULE: 今日打卡

### Requirement: 今日发疯打卡
系统 SHALL 提供"今日发疯打卡"功能。

#### Scenario: 用户状态不好需要急救
- **WHEN** 用户点击"今日发疯打卡"
- **THEN** 记录打卡时间，显示急救工具箱入口

### Requirement: 打卡记录展示
系统 SHALL 展示用户的打卡历史记录。

#### Scenario: 用户查看打卡记录
- **WHEN** 用户进入急救页面
- **THEN** 显示今日是否已打卡，以及近期打卡记录

## MODULE: 急救工具箱

### Requirement: 紧急联系人
系统 SHALL 提供一键拨打紧急联系人的功能。

#### Scenario: 用户需要紧急求助
- **WHEN** 用户点击"紧急联系人"
- **THEN** 显示预设联系人列表，支持一键拨打

### Requirement: 解压小游戏
系统 SHALL 提供在线解压小游戏。

#### Scenario: 用户需要发泄
- **WHEN** 用户点击"解压一下"
- **THEN** 跳转至解压小游戏页面

### Requirement: 专业咨询入口
系统 SHALL 提供心理咨询热线/预约入口。

#### Scenario: 用户需要专业帮助
- **WHEN** 用户点击"专业咨询"
- **THEN** 显示心理援助热线和咨询预约入口

## 数据结构

### CheckinRecord
```typescript
interface CheckinRecord {
  id: string
  date: string          // 打卡日期 YYYY-MM-DD
  time: string          // 打卡时间 HH:mm
  mood?: number         // 心情指数 1-5
  note?: string         // 简短备注
  triggered: boolean    // 是否触发过急救
}
```

```typescript
interface EmergencyContact {

  name: string

  relationship: string
  isPrimary: boolean    // 是否为主要联系人
}
```

## 状态定义

### 打卡状态
| 状态 | 描述 | 视觉表现 |
|------|------|----------|
| 未打卡 | 今天还没打卡 | 灰色/等待中 |
| 已打卡 | 今天已打卡求急救 | 红色闪烁/警示 |
| 急救中 | 正在使用急救工具 | 脉冲动画 |

### 心情指数
| 指数 | 描述 | 颜色 |
|------|------|------|
| 1 | 很糟糕 | 深红 |
| 2 | 不太好 | 橙红 |
| 3 | 一般 | 黄色 |
| 4 | 还行 | 浅绿 |
| 5 | 很好 | 绿色 |

## REMOVED Requirements
- 删除原有的"发疯文案生成器"功能（用户输入场景生成文案）
- 删除 quickScenarios 快捷场景
- 删除 contentLibrary 文案库

## API 接口

### GET /api/rescue/checkinToday
获取今日打卡状态

### POST /api/rescue/checkin
提交今日打卡
```json
{
  "mood": 2,
  "note": "被领导骂了"
}
```

### GET /api/rescue/checkinHistory
获取打卡历史记录

### GET /api/rescue/contacts
获取紧急联系人列表

### POST /api/rescue/contacts
添加紧急联系人
