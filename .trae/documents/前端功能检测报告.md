## 二、严重问题详情

### 问题1: 邮箱验证正则表达式错误

**位置**: [pages/profile/edit.vue:204](file:///c:/Users/Administrator/Desktop/3/pages/profile/edit.vue#L204)

**问题描述**:
```typescript
const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/
```
正则表达式中使用了错误的转义字符 `\s` 应该是 `\s` (空白字符),但实际写成了 `\s` (字面量s),导致邮箱验证功能失效。

**影响**: 
- 用户无法正确验证邮箱格式
- 可能导致无效邮箱被提交到后端

**修复建议**:
```typescript
const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/
```

---

### 问题2: ResponseData类型未定义

**位置**: [api/mood.ts:17](file:///c:/Users/Administrator/Desktop/3/api/mood.ts#L17)

**问题描述**:
```typescript
export const calculateMoodIndex = (
  data: MoodCalculateRequest
): Promise<ResponseData<MoodCalculateResponse>> => {
  return post('/api/mood/calculate', data)
}
```
使用了 `ResponseData` 泛型类型,但该类型未在文件中定义,也未从其他文件导入。

**影响**:
- TypeScript编译错误
- 类型检查失效
- 可能导致运行时错误

**修复建议**:
```typescript
// 在 api/mood.ts 文件顶部添加类型定义
interface ResponseData<T = any> {
  code: number
  message: string
  data: T
}
```

### 问题4: 腾讯地图API密钥硬编码为占位符

**位置**: [pages/rescue/index.vue:202](file:///c:/Users/Administrator/Desktop/3/pages/rescue/index.vue#L202)

**问题描述**:
```typescript
const geoRes = await uni.request({
  url: `https://apis.map.qq.com/ws/geocoder/v1/?location=${locRes.latitude},${locRes.longitude}&key=YOUR_KEY`,
  method: 'GET'

export const TENCENT_MAP_KEY = 'your_actual_api_key'

// 2. 在 pages/rescue/index.vue 中使用
import { TENCENT_MAP_KEY } from '@/utils/config'
const geoRes = await uni.request({
  url: `https://apis.map.qq.com/ws/geocoder/v1/?location=${locRes.latitude},${locRes.longitude}&key=${TENCENT_MAP_KEY}`,
  method: 'GET'
})
```

### 问题4: 头像显示逻辑问题

**位置**: [pages/mine/index.vue:223-224](file:///c:/Users/Administrator/Desktop/3/pages/mine/index.vue#L223-L224)

**问题描述**:
```vue
<image v-if="userInfo.avatar" :src="userInfo.avatar" class="avatar-img" mode="aspectFill" />
<text v-else class="avatar-text">{{ userInfo.nickname ? userInfo.nickname.charAt(0) : '醒' }}</text>
```
当 `userInfo.avatar` 为空字符串 `''` 时,`v-if="userInfo.avatar"` 仍会返回 `true`,导致显示空的image标签。

**影响**:
- 头像显示异常
- 可能显示空白区域
- 用户体验差

**修复建议**:
```vue
<image v-if="userInfo.avatar && userInfo.avatar.trim()" :src="userInfo.avatar" class="avatar-img" mode="aspectFill" />
<text v-else class="avatar-text">{{ userInfo.nickname ? userInfo.nickname.charAt(0) : '醒' }}</text>
```
