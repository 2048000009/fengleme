# 疯情指数页面重构计划

## 背景

* 首页（`pages/index/index.vue`）已有打卡功能，记录用户的签到数据（`checkin_data`）

* 目前的"疯情指数"页面（`pages/mood/index.vue`）是问答测试模式

* 用户要求反向逻辑：**打卡 = 发疯**，用打卡数据来展示用户的疯情指数

## 核心理念转变

* **旧逻辑**：做测试题 → 得到疯批等级（问答式）

* **新逻辑**：打卡积累疯度 → 显示发疯程度（数据驱动）

## 重构方案

### 1. 数据层改动

* 复用 `utils/storage.ts` 中的 `CHECKIN_KEY = 'checkin_data'`

* 从 storage 读取用户的打卡数据，包含：

  * `lastDate`: 最后签到日期

  * `continuousDays`: 连续签到天数

  * `totalDays`: 累计签到天数

  * `crazyPoints`: 总疯度

  * `joinDate`: 加入日期

### 2. 页面结构改动

#### 2.1 头部区域

* 显示用户等级称号（如"疯神降临"、"疯林盟主"等）

* 基于连续签到天数和总疯度计算

* 使用首页已有的 `crazyLevel` 逻辑

#### 2.2 疯情指数卡片（核心）

* 大数字显示"疯度"总数

* 环形/条形进度条显示与下一等级的差距

* 可视化展示发疯程度

#### 2.3 统计概览

* 连续签到天数

* 累计签到天数

* 今日获得疯度

* 最近签到时间分析（判断是早起疯还是夜猫子疯）

#### 2.4 发疯日历/时间分布

* 可选：展示最近打卡的日期分布

* 分析打卡时间段（凌晨/早上/下午/深夜），给出"发疯类型"标签

  * 例如：

    * "深夜修仙型" - 经常凌晨打卡

    * "早起奋斗型" - 早上打卡

    * "佛系随缘型" - 时间不固定

#### 2.5 发疯语录/状态展示

* 根据用户的发疯程度，展示不同的趣味语录

* 与首页打卡语录风格一致

### 3. 交互改动

* 移除：场景选择 → 问答流程 → 结果展示

* 新增：数据可视化展示 + 打卡记录入口

* 保留："重新测试"概念 → 变成"重新开始记录"（跳转到首页打卡）

### 4. 样式调整

* 保持现有配色（#00D4AA 主题色）

* 使用卡片式布局展示各项数据

* 添加动画效果让数据展示更生动
